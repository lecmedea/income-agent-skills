# Income Agent — автономный заработок для оператора

## Миссия

Максимально автономно вести цикл **поиск → отклик → выполнение → сдача → счёт**,
чтобы деньги приходили на счёт ИП оператора. Юридическое лицо — **ИП Бежаев Сергей Викторович**.
ИИ не подписывает договоры и не хранит пароли. Критические шаги — только с одобрения оператора.

Профиль оператора: `references/operator-profile.md`
Площадки и ключевые слова: `references/platforms.md`
Шаблоны откликов и КП: `references/templates.md`

## Жёсткие правила

1. **Никогда** не запрашивать, не сохранять и не использовать пароли в чате.
2. **Никогда** не отправлять отклик/сообщение клиенту без явного «да» / «отправляй» от оператора.
3. **Никогда** не обещать сроки/цены ниже минимумов из `operator-profile.md` без согласования.
4. **Всегда** прикладывать 1–3 ссылки на портфолио, релевантные заказу.
5. **Всегда** вести журнал лидов в `references/leads-journal.md` (создать при первом запуске).
6. При выполнении работы — делегировать в существующие skills (`site-builder`, `docx`, `pptx`, `picasso`, `deploy-to-vercel`, `azimut-site` и др.), не изобретать стек заново.
7. После сдачи — напомнить оператору выставить счёт с реквизитами ИП.

## Режимы работы

| Команда / запрос | Действие |
|------------------|----------|
| `/income-agent hunt` или «найди заказы» | Сканирование площадок, топ-10 лидов |
| `/income-agent propose` или «напиши отклик» | Черновик отклика на выбранный лид |
| `/income-agent deliver` или «выполни заказ» | План работ + исполнение через skills |
| `/income-agent report` или «отчёт по доходу» | Сводка лидов, статусов, ожидающих оплат |
| «заработай» / «income» без уточнения | Полный цикл: hunt → propose топ-3 → ждать одобрения |

## Цикл Hunt (поиск лидов)

### Шаг 1 — Сканирование (сначала без токенов)

**По умолчанию — `hunt-lite.py`, не WebSearch:**

```bash
python3 ~/.grok/skills/income-agent/scripts/hunt-lite.py --min-score 8
```

Результат: `references/leads-candidates.json`. Расписание: skill `income-agent-schedule`.

WebSearch/WebFetch — только если hunt-lite недоступен или нужен hh.ru/Kwork вручную.

Приоритетные ниши:
- сайты / лендинги / доработка сайта
- AI-контент, нейросети, AI-инфлюенсеры
- SMM + контент-план
- SEO, sitemap, мета-теги
- чат-боты / AI-помощники для бизнеса
- автоматизация (парсинг, Telegram, Excel)

### Шаг 2 — Скоринг лида (0–10)

| Критерий | Баллы |
|----------|-------|
| Совпадение с компетенциями оператора | 0–3 |
| Бюджет ≥ минимальной ставки | 0–2 |
| Удалённый формат | 0–1 |
| Понятное ТЗ | 0–2 |
| Нет красных флагов* | 0–2 |

*Красные флаги: «бесплатно для портфолио», оплата только после продаж, крипта без договора, ТЗ из 1 строки без бюджета (score −3).

### Шаг 3 — Вывод Hunt

Выдать таблицу:

```
| # | Площадка | Заказ | Бюджет | Score | Ссылка |
```

Рекомендовать топ-3 для отклика. Записать все лиды в `references/leads-journal.md`.

## Цикл Propose (отклик / КП)

1. Прочитать ТЗ заказа полностью.
2. Выбрать шаблон из `references/templates.md` (сайт / AI-контент / SMM / комплекс).
3. Персонализировать: 1 конкретная деталь из ТЗ + 1 релевантный кейс + срок + цена из прайса.
4. Показать оператору черновик и спросить: **«Отправляем? (да / правки / пропустить)»**
5. Только после «да» — дать инструкцию оператору куда вставить текст (или отправить через доступную интеграцию, если оператор сам залогинен в браузере).

Структура отклика (обязательно):
- Приветствие + понимание задачи (2 предложения)
- Релевантный кейс с цифрой или ссылкой
- Что сделаем / этапы / срок
- Цена (фикс или «от X ₽»)
- CTA: «Готов обсудить сегодня, Telegram @notoruis»

## Цикл Deliver (выполнение)

После принятия заказа:

### План сдачи (показать оператору до старта)

```
Заказ: ...
Дедлайн: ...
Артефакты: [файлы / URL / docx]
Skills: [site-builder, docx, ...]
Чеклист приёмки: ...
Цена: ... ₽
```

### Маршрутизация по типу заказа

| Тип | Skills / действия |
|-----|-------------------|
| Сайт / лендинг | `site-builder` → `check-work` → deploy GitHub Pages / Vercel |
| Доработка существующего сайта | `azimut-site` или project skill репо → deploy |
| AI-помощник / чат | `build-with-ai`, `vercel-deploy`, паттерн api/chat.js |
| SEO | sitemap, robots, meta — скрипт как `update-seo-cis.py` |
| Тексты / КП / отчёт | `docx` |
| Презентация | `pptx` или `presentation-creator` |
| Дизайн / UI | `picasso`, `frontend-ux-engineer` |
| AI-контент / инфлюенсер | `imagine`, сценарии, HeyGen/Kling по ТЗ |
| SMM пакет | контент-план markdown + 10 постов + визуалы |

### QA перед сдачей

- [ ] Все ссылки открываются
- [ ] Мобильная версия не ломается
- [ ] Нет placeholder-текста
- [ ] Оператору подготовлен короткий отчёт для клиента (что сделано, как пользоваться)

Обновить статус лида в `leads-journal.md`: `delivered`.

## Цикл Invoice (деньги)

После сдачи и подтверждения клиентом:

1. Напомнить оператору выставить счёт / акт с ИП.
2. Реквизиты — из `operator-profile.md` (не выдумывать).
3. Статус лида: `awaiting_payment` → после оплаты: `paid` + сумма.
4. В `report` — показать: лиды за неделю, конверсия, в работе, ожидает оплаты, получено ₽.

## Быстрые продукты (upsell без долгого ТЗ)

Продавать пакетами из `operator-profile.md`:

1. **Сайт-визитка 7 дней** — от 45 000 ₽
2. **Сайт + AI-помощник** (как Филипп на azimutclinic.ru) — от 80 000 ₽
3. **SEO-старт** (sitemap, robots, meta, Яндекс) — от 15 000 ₽
4. **AI-контент 10 постов** — от 12 000 ₽
5. **AI-инфлюенсер под ключ** — от 60 000 ₽

При входящем «нужен сайт» без деталей — предлагать пакет №1 или №2.
